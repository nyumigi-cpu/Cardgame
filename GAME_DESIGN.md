# 씹덕 일러스트 카드게임 기획서

## 프로젝트 개요
애니풍 일러스트 카드 배틀 게임 — 웹 기반, 모바일 최적화

---

## 듀얼 채팅 개발 체계
모바일에서 Opus/Sonnet 모델 선택이 불가한 제약을 우회하기 위해,
**두 개의 채팅창을 동시에 운용**하며 이 기획서를 공유 기준점으로 사용한다.

| 채팅창 | 모델 | 역할 |
|--------|------|------|
| **Chat A** | Opus | 핵심 로직, 아키텍처, 복잡한 게임 시스템 |
| **Chat B** | Sonnet | UI 컴포넌트, 스타일링, 반복 작업 |

### 작업 동기화 규칙
1. 모든 작업은 이 기획서(`GAME_DESIGN.md`)를 기준으로 진행
2. 각 채팅에서 작업 완료 시 기획서의 체크리스트를 업데이트
3. 충돌 방지: 같은 파일을 동시에 수정하지 않도록 역할 분리

## 게임 컨셉

### 장르
- 턴제 카드 배틀 (TCG 라이트)
- 1v1 PvP / AI 대전

### 테마 & 아트 스타일
- 애니/만화풍 일러스트
- 캐릭터 카드 중심 (미소녀/미소년 + 판타지)
- 속성 시스템: 🔥 화 / 💧 수 / 🌿 목 / ⚡ 뇌 / 🌙 암 / ☀️ 광

## 핵심 시스템

### 카드 구조
```
{
  id: string,
  name: string,           // 카드 이름
  illustration: string,   // 일러스트 경로
  rarity: "N" | "R" | "SR" | "SSR" | "UR",
  element: "fire" | "water" | "wood" | "thunder" | "dark" | "light",
  type: "character" | "spell" | "trap",
  cost: number,           // 소환/사용 코스트 (1~10)
  atk: number,            // 공격력 (캐릭터만)
  hp: number,             // 체력 (캐릭터만)
  skill: {
    name: string,
    description: string,
    effect: SkillEffect
  },
  flavor_text: string     // 카드 뒷이야기
}
```

### 배틀 규칙
- 각 플레이어: 30 HP, 덱 30장
- 턴 시작 시 카드 1장 드로우
- 매 턴 코스트 +1 (최대 10) — 마나 시스템
- 필드에 캐릭터 최대 4체 배치
- 속성 상성:
  - 화 > 목 > 뇌 > 수 > 화
  - 암 ↔ 광 (상호 강)
- 상성 유리: 데미지 x1.5 / 불리: 데미지 x0.75

### 승리 조건
- 상대 HP를 0으로 만들기
- 상대 덱이 0장일 때 드로우 불가 시 패배

| 영역 | 기술 |
|------|------|
| 프론트엔드 | HTML/CSS/JS (바닐라 or React) |
| 게임 렌더링 | Canvas API 또는 CSS 기반 |
| 백엔드 | 추후 결정 (일단 로컬 싱글플레이) |
| 카드 데이터 | JSON 파일 |
| 일러스트 | placeholder → 추후 AI 생성 or 직접 제작 |

## 카드 예시

### SSR — 화염의 검사 아카네
- 속성: 🔥 화
- 코스트: 5 | ATK: 7 | HP: 5
- 스킬: **블레이즈 슬래시** — 적 캐릭터 1체에 ATK+2 데미지, 목 속성이면 추가 3 데미지
- *"불꽃은 내 검에서 태어나 적의 잿더미 위에서 잠든다."*

### SR — 수호의 무녀 미즈키
- 속성: 💧 수
- 코스트: 4 | ATK: 3 | HP: 8
- 스킬: **치유의 물결** — 아군 전체 HP 2 회복
- *"이 물은 상처를 씻고, 마음을 채워줍니다."*

### R — 번개 고양이 라이냥
- 속성: ⚡ 뇌
- 코스트: 2 | ATK: 4 | HP: 3
- 스킬: **스파크 러시** — 소환 즉시 적 캐릭터 1체에 2 데미지
- *"냥냥! 찌릿찌릿하다냥~!"*
